# CrowBar v0.1.10

Fixes CrowBar rendering on remote servers without Allium, NPC UUIDs showing in the locator bar, and XP bar visibility.

## Fixed
- **Locator bar respects vanilla visibility**: CrowBar now checks whether the vanilla locator bar is actually present before rendering its custom background and addons (skins, nametags, distance text). On remote servers without Allium where the vanilla locator bar is not showing, CrowBar stays completely hidden instead of drawing its own background into empty space.
- **Locator bar hidden when experience is gained**: When the player gains experience, CrowBar's entire renderer (background + addons) disappears for 5 seconds — matching the vanilla locator bar's own hide duration — so the vanilla XP bar is fully visible. CrowBar reappears automatically once the 5 seconds pass.
- **NPC UUIDs filtered from locator bar**: The client-level player fallback now verifies each entry has a valid player list entry, excluding server-side NPCs (e.g. Citizens) that previously showed as raw UUID strings.
- **State cleared on server/world switch**: All CrowBar state (Allium data, integrated server flag) is properly reset on both JOIN and DISCONNECT events, preventing stale renderer state from carrying over when transferring between servers (e.g. lobby to survival).
- **v21 debug logging removed**: Removed stray `System.out.println` calls from the dot color renderer.
