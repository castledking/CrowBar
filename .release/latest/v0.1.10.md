# CrowBar v0.1.10

Fixes CrowBar rendering on remote servers without Allium and NPC UUIDs showing in the locator bar.

## Fixed
- **Locator bar no longer replaces vanilla on remote servers without Allium**: CrowBar only draws its custom background when Allium data is present or on integrated servers. On remote servers without Allium, vanilla renders its locator bar normally and CrowBar overlays skins, nametags, and distance text on top.
- **NPC UUIDs filtered from locator bar**: The client-level player fallback now verifies each entry has a valid player list entry, excluding server-side NPCs (e.g. Citizens) that previously showed as raw UUID strings.
- **State cleared on server/world switch**: All CrowBar state (Allium data, integrated server flag) is properly reset on both JOIN and DISCONNECT events, preventing stale renderer state from carrying over when transferring between servers (e.g. lobby to survival).
- **v21 debug logging removed**: Removed stray `System.out.println` calls from the dot color renderer.
