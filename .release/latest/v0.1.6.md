# CrowBar v0.1.6

Minecraft 1.21.6+ and 26.1.x support

## Added

- **Minecraft 26.1.x jar**: CrowBar now ships a dedicated `CrowBar-26.1.x` build alongside the `CrowBar-1.21.x` build.
- **Allium-backed locator rendering**: when an Allium server is present, CrowBar renders from Allium's player payload instead of depending on vanilla locator waypoints.
- **Distance text**: `X` toggles distance labels on the locator bar.
- **LAN fallback for 26.1.x**: non-Allium LAN and integrated-server sessions fall back to client-visible players.

## Changed

- **1.21.x compatibility range**: the 1.21 jar now targets Minecraft 1.21.6 through 1.21.11.
- **26.1.x renderer path**: locator background, dots, player heads, name tags, distance labels, and team colors now render through CrowBar's HUD renderer.
- **Player heads on 26.1.x**: skin rendering mirrors vanilla's player-list face extraction path.
- **Keybind category handling on 26.1.x**: CrowBar controls now appear under one CrowBar category.
- **Allium session reset on 26.1.x**: Allium state is cleared on join and disconnect, so connecting to an Allium server cannot prime the renderer for the next LAN session.

## Removed

- **Force show all**: removed the unused config option, keybind, state, language string, and renderer fallback.

## Requirements

- `CrowBar-1.21.x`: Minecraft 1.21.6-1.21.11, Fabric Loader 0.18.4 or newer, Fabric API 0.141.4 or newer, Java 21
- `CrowBar-26.1.x`: Minecraft 26.1.2 or newer in the 26.1.x line, Fabric Loader 0.19.2 or newer, Fabric API 0.150.0 or newer, Java 25
- Allium is optional, but recommended for Paper servers that need hidden, sneaking, vanished, invisible, or ViaVersion-connected players to appear correctly on CrowBar's locator bar.
