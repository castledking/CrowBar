# CrowBar v0.1.7

Allium locator bar polish and stale-entry fixes.

## Added
- **Allium vertical arrows**: shows vanilla-styled up/down arrows for nearby Allium targets within 48 blocks when the locator dot is visible but the target is outside the viewer's vertical camera angle.
- **Arrow sprite assets**: packages the vanilla two-frame arrow sprites through the GUI sprite atlas for both 1.21.x and 26.1.x builds.

## Changed
- **Dot sprite distance bands**: Allium-rendered dots now use vanilla-style sprite thresholds: `0-128`, `129-230`, `231-332`, and `333+` blocks.
- **Arrow placement**: up arrows render above the dot and down arrows render below the dot.
- **Allium snapshot handling**: CrowBar now treats Allium payloads as complete snapshots, so omitted, disconnected, and cross-world players are removed immediately.
- **Allium renderer scope**: Allium vertical arrows only render while the target's dot/name is inside the locator bar's yaw window.
- **26.1.x Allium/LAN separation**: Allium state is cleared on join/disconnect and LAN fallback remains available outside Allium sessions.

## Fixed
- Fixed vertical arrow range so arrows render only within 48 blocks instead of outside 48 blocks.
- Fixed raw UUID/dot leftovers when an Allium player leaves by replacing stale snapshot data.
- Fixed cross-world Allium entries lingering on the locator bar after server-side payload changes.
- Fixed untextured arrow rendering by moving arrow assets into the GUI sprite path.
